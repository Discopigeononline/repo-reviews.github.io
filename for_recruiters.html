For Recruiters

Recruiters use Repo Reviews as a screening exercise for candidates to demonstrate familiarity with GitHub while showcasing a thougthful review that benefits the open-source community.

GitHub Fork

Code collaboration and review are built into the development process with GitHub. Candidates start by creating a GitHub account and forking the repo-reviews repo.

Edit YAML & Markdown

Candidates edit the YAML front matter of a review template and create a thoughtful review of a repo of their choice using Markdown.

GitHub Pull Requests

Candidates issue a GitHub pull-request (PR) to merge their review into repo-reviews.

GitHub Actions & CI

Candidates watch as a GitHub action validates their PR and Continuous Integration incorporates their review into the website.

GitHub Issues

Candidates are tagged in an auto-created issue and prompted to share their review with the repo.

Contribute to the community

Candidates experience the joy of contributing to an open-source repo and sharing their experience with the community to increase productivity.
